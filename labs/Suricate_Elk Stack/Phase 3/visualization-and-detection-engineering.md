Phase 3 — Visualization & Detection Engineering

## Objective
Build visualization and analysis capability in Kibana, organize Suricata alert data into meaningful views, and begin tuning detections to improve signal-to-noise ratio.

---

## Outcome

- Created Kibana index patterns for Suricata data
- Validated searchable security events in Kibana Discover
- Built initial visualizations for alerts, source IPs, destination IPs, and signatures
- Improved understanding of how detections are interpreted in a SOC workflow
- Began tuning detection logic by identifying noisy and high-value alerts

---

# 🧱 Architecture

```text
Traffic → Suricata
              ↓
        eve.json logs
              ↓
        Logstash
              ↓
        Elasticsearch
              ↓
        Kibana Discover / Dashboards / Visualizations
🛠️ Implementation Steps
1. Confirm Phase 2 Pipeline Health

Verified all services were running:

sudo systemctl status suricata
sudo systemctl status elasticsearch
sudo systemctl status logstash
sudo systemctl status kibana

Confirmed Elasticsearch indices existed:
curl 'http://localhost:9200/_cat/indices?v'

Result:

Verified Suricata indices were being created and populated
2. Access Kibana

Opened Kibana in browser:

http://10.10.30.100:5601

Result:

Confirmed Kibana web interface was reachable from the lab management network
3. Create / Verify Index Pattern

Created or verified index pattern:

suricata-*

Selected time field:

@timestamp

Result:

Enabled searching and filtering of Suricata events in Kibana Discover
4. Validate Data in Discover

Reviewed events in Kibana Discover and confirmed the presence of:

src_ip
dest_ip
alert.signature
event_type
@timestamp

Filtered for alert events only.

Result:

Confirmed Suricata alerts were being indexed and displayed properly
5. Generate Fresh Test Traffic

From Kali, generated additional scan traffic:

nmap -sS -T4 -A 10.10.30.100

Optional expanded testing:

nmap -p- 10.10.30.100
nmap -sV 10.10.30.100

Result:

Created fresh events for validation and dashboard building
6. Build Basic Visualizations

Created initial Kibana visualizations for:

Alerts over time
Top source IP addresses
Top destination IP addresses
Top alert signatures
Event count by type

Result:

Converted raw log events into analyst-friendly visual views
7. Build Initial Dashboard

Created a dashboard combining:

Alert timeline
Top signatures
Top source IPs
Top destination IPs

Result:

Established a basic SOC-style monitoring dashboard for Suricata alert data
8. Identify High-Value vs Noisy Alerts

Reviewed recurring signatures to determine:

Which alerts represented meaningful scan activity
Which alerts were repetitive or lower-priority
Which detections aligned to intentional attack simulation

Result:

Began separating useful detections from expected background noise
9. Begin Detection Tuning

Documented candidate tuning areas, including:

Repeated scan signatures
Internal lab noise
Alerts generated by expected test traffic

Tuning actions considered:

Rule suppression
Threshold tuning
Signature review

Result:

Began transition from raw detection to detection engineering
10. Validate End-to-End Workflow

Confirmed the full workflow:

Attack Simulation → Detection → Log Ingestion → Search → Visualization → Analysis

Result:

Validated that the environment supports both detection and analyst workflow
💥 Summary

Phase 3 successfully established:

Searchable alert review in Kibana
Basic SOC-style visualizations
Initial dashboard creation
Detection interpretation and early tuning
Transition from raw logs to operational visibility
